# Отчёт о подготовке сервера

## Основная информация

* Назначено имя сервера `web-prod-01`.
* Создан пользователь `deploy` с домашним каталогом `/home/deploy`.
* Для пользователя `deploy` назначена командная оболочка `/bin/bash`.
* Пользователь `deploy` добавлен в административную группу `sudo`.
* В `/etc/sudoers.d/deploy` настроен запуск `sudo` без пароля через правило `NOPASSWD`.
* Для файла `/etc/sudoers.d/deploy` установлены права `440`.

## Настройка SSH

* Для пользователя `deploy` создан каталог `/home/deploy/.ssh`.
* Публичный SSH-ключ добавлен в `/home/deploy/.ssh/authorized_keys`.
* Для каталога `.ssh` установлены права `700`.
* Для файла `authorized_keys` установлены права `600`.
* Владельцем SSH-файлов назначен пользователь `deploy`.
* Вход по паролю отключён директивой `PasswordAuthentication no`.
* Прямой вход под пользователем `root` отключён директивой `PermitRootLogin no`.
* Конфигурация SSH проверена командой `sshd -t`.

## Firewall

* Установлена политика UFW `deny incoming` для входящих подключений.
* Установлена политика UFW `allow outgoing` для исходящих подключений.
* Разрешён порт `22/tcp` для SSH.
* Разрешён порт `80/tcp` для HTTP.
* Разрешён порт `443/tcp` для HTTPS.
* Firewall UFW активирован.

## Обновления и время

* Списки пакетов обновлены с помощью `apt-get update`.
* Установлены доступные обновления с помощью `apt-get upgrade -y`.
* Служба синхронизации времени `chrony` включена и активна.
* Синхронизация времени проверена командой `chronyc tracking`.

## Swap

* Создан swap-файл `/swapfile` размером 256 МБ.
* Для `/swapfile` установлены права `600`.
* Файл подготовлен командой `mkswap`.
* В `/etc/fstab` добавлена запись для автоматического подключения swap.
* В учебной среде реальное включение swap недоступно из-за ограничений контейнера.

## Fail2ban

* Установлен пакет `fail2ban`.
* Создан файл конфигурации `/etc/fail2ban/jail.local`.
* Включён jail `[sshd]`.
* Установлен лимит `maxretry = 5`.
* Время блокировки задано параметром `bantime = 1h`.
* В учебной среде демон не запускается из-за отсутствия полноценного netfilter-бэкенда.

## Мониторинг диска

* Создан исполняемый скрипт `/opt/ops/disk_watch.sh`.
* Скрипт проверяет заполненность корневого раздела `/`.
* При заполнении диска более чем на 80% скрипт отправляет предупреждение в системный журнал через `logger`.
* В crontab пользователя `deploy` добавлен запуск скрипта каждые пять минут.
